Origen y propósito de C

  • C es un lenguaje de programación de propósito general desarrollado junto con UNIX.
  • Aunque está estrechamente ligado a UNIX, es independiente de cualquier sistema operativo o máquina.
  • Inicialmente diseñado para programación de sistemas, también se utiliza con éxito en aplicaciones numéricas, de texto y bases de datos.

Características del lenguaje

  • C se considera un lenguaje de “bajo nivel” porque trabaja directamente con datos básicos como caracteres, números y direcciones de memoria.
  • No incluye operaciones integradas para estructuras complejas ni administración automática de memoria, lo que lo hace simple y eficiente.

Administración de memoria

  • C otorga al programador control total sobre la memoria mediante las funciones malloc() y free().
  • Esta flexibilidad permite programas eficientes, pero con riesgos como fugas de memoria y fragmentación, problemas que lenguajes modernos (Java, Python) resuelven con recolección automática de basura.

Simplicidad y portabilidad

  • C omite deliberadamente muchas características de alto nivel (como la recolección de basura o la programación paralela) para mantener su velocidad y simplicidad.
  • Gracias a esto, los compiladores son fáciles de construir y los programas escritos en C son altamente portables entre diferentes máquinas.

Eficiencia y UNIX

  • Los programas en C son tan eficientes que reemplazan al lenguaje ensamblador en la mayoría de los casos.
  • El sistema operativo UNIX fue escrito casi por completo en C, demostrando su rendimiento y portabilidad en diversas plataformas de hardware.

Introducción al lenguaje C

  • La mejor forma de aprender C es escribiendo programas. El clásico primer programa imprime en pantalla el texto:
#include <stdio.h>
main() {
printf("hello, world\n");
}
  • Este ejemplo enseña cómo compilar y ejecutar un programa (por ejemplo, con gcc -ansi hello.c y luego ./a.out) y demuestra la estructura básica de un programa en C.

Introducción al lenguaje C

  • Un programa en C se compone de una o más funciones, siendo main() el punto de inicio obligatorio.
  • Los bloques de código se encierran entre llaves { }, y los paréntesis () se utilizan para definir los argumentos de una función.
La función printf() se usa para mostrar texto en la terminal.
#include <stdio.h>
main() {
printf("hello, world\n");
}

Cadenas y secuencias de escape

  • Las cadenas de caracteres se escriben entre comillas dobles " ". C utiliza secuencias de escape para representar caracteres especiales:
  • \n → nueva línea
  • \t → tabulación
  • \\ → barra invertida
  • \" → comillas dobles
  • Estas secuencias permiten controlar el formato de salida en pantalla.

Variables y tipos de datos

  • Antes de usarse, todas las variables deben declararse con su tipo de dato:
int lower, upper, step;
float fahr, celsius;
  • int representa números enteros.
float representa números con decimales. La precisión y tamaño dependen del hardware; por ejemplo, en el PDP-11, un int tiene 16 bits y un float 32 bits.

Ejemplo práctico: Conversión de temperatura

  • Crear un programa en C que genere la siguiente tabla de temperaturas en Fahrenheit y las convierta a Celsius usando la fórmula C=(5/9)(F-32)

Ejemplo práctico: Conversión de temperatura

1. #include <stdio.h> Incluye la biblioteca estándar que contiene la función printf() usada para mostrar texto en pantalla.
  • 2. Comentario entre /* ... */ Describe brevemente la función del programa. Los comentarios son ignorados por el compilador.
  • 3. Declaración de variables:
int lower, upper, step;
float fahr, celsius;
int: números enteros (valores sin decimales).
  • float: números con decimales. Las variables deben declararse antes de usarse.

Ejemplo práctico: Conversión de temperatura

  • 4. Asignaciones iniciales:
  • lower = 0; // Límite inferior
  • upper = 300; // Límite superior
  • step = 20; // Incremento entre mediciones
  • fahr = lower; // Valor inicial
  • Estas líneas definen el rango de temperaturas a convertir.
  • 5. Bucle while:
while (fahr <= upper) {
  • ...
}
  • Ejecuta repetidamente el bloque entre llaves mientras la condición sea verdadera. Cada iteración calcula y muestra una fila de la tabla.

Ejemplo práctico: Conversión de temperatura

  • 6. Cálculo de la conversión:
celsius = (5.0/9.0) * (fahr - 32.0);
  • Convierte Fahrenheit a Celsius. Se usan valores con punto decimal (5.0, 32.0) para evitar división entera, que truncaría los decimales.
  • 7. Impresión formateada:
printf("%4.0f %6.1f\n", fahr, celsius);
  • %4.0f: imprime el primer número (Fahrenheit) con 4 espacios y 0 decimales.
  • %6.1f: imprime el segundo número (Celsius) con 6 espacios y 1 decimal.
  • \n: salto de línea.
  • 8. Incremento del valor:
fahr = fahr + step;
  • Aumenta la temperatura en 20 unidades hasta alcanzar el límite superior.

Ejemplo práctico: Conversión de temperatura

  • Comentarios (/* ... */) ayudan a documentar el código.
  • Indentación y llaves { } indican bloques lógicos.
printf() formatea la salida; cada % se asocia a una variable.
  • Tipos de datos (int, float, char, double) determinan cómo se almacenan los valores.
  • Evitar divisiones enteras: siempre usar constantes con punto decimal en operaciones de punto flotante.

Ciclo For

  • Es el mismo resultado que con while, pero más compacto: inicialización, condición y actualización juntas.

Ciclo For

  • Qué hace cada parte:
int fahr; → variable de control (entera) para mostrar %d.
for (ini; cond; act) → fahr=0 (una vez), fahr<=300 (condición), fahr=fahr+20 (paso).
printf(...) → imprime Fahrenheit y Celsius; la expresión se pone directo como 3er argumento. Cuándo usar for vs while: usa for cuando ini/cond/actualización están lógicamente relacionadas.

Constantes

  • Nombres en MAYÚSCULAS para distinguir constantes.
  • Sin punto y coma al final del #define.
  • Facilita mantenimiento y lectura.

Entrada y Salida de caracteres

  • Concepto: I/O no es parte del lenguaje, sino de la librería estándar (stdio.h).
c = getchar(); → lee siguiente carácter (o EOF al final).
putchar(c); → escribe un carácter.
  • Detalles importantes:
  • EOF está definido en stdio.h (no lo redefinas).
  • Utiliza CTRL-D (unix-style) or CTRL-Z (Windows) para enviar el EOF

Arreglos

  • Es un bloque contiguo de memoria con N elementos del mismo tipo.
  • El índice va de 0 a N−1 (convención C).
  • No hay verificación de límites: si te sales del rango, existe un comportamiento indefinido, generalmente conocido como "desbordamiento de búfer", donde un programa lee más datos de los que caben en un arreglo y es engañado para sobrescribir otros datos o código y comprometer una aplicación.

Diapositiva 21

sizeof(a) en un arreglo local devuelve tamaño en bytes de todo el arreglo.
sizeof(a)/sizeof(a[0]) es el número de elementos (técnica clásica).
  • OJO: cuando un arreglo se pasa a función, “se degrada” a puntero y sizeof ya no da el total, sino el tamaño del puntero.

Arreglos

  • Si no inicializas, el contenido es indefinido (basura) para arreglos automáticos (dentro de una función).
  • Inicialización parcial rellena con ceros lo que falte.
  • = {0} forza todos a cero.

Contar Palabras

  • inword → estado (estoy dentro de una palabra o no).
  • nl = nw = nc = 0; → asignaciones encadenadas (asocian derecha→izquierda).
  • || y && → OR/AND con cortocircuito (deja de evaluar cuando ya sabe el resultado).
